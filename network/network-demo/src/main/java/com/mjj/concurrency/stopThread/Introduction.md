Java提供了stop() suspend() resume() 三个方法来停止、挂起、恢复线程，但是这三个方法
都已经被明确的被废弃，因为他们存在不小的安全隐患：
    1. suspend()方法，在导致线程暂停的时候，并不释放任何资源，若其他线程也想访问它占用的
       资源，也会受到影响导致无法运行。
       
    2. resume()方法，用于恢复被suspend挂起的程序，但是如果resume在suspend之前运行了，
       那就会导致挂起的线程继续挂起，它占用锁也不会释放，会导致整个系统无法正常工作
    
    3. stop()方法，会简单粗暴的停止线程，可能导致线程无法正确释放资源。
    
    
如何安全的停止线程呢?
首先明确的是  Java的线程机制是协作式的，而非抢占式，只是通过interrupt()方法通知线程中断，
将该线程的中断标志位置为true，具体实现有被通知的线程自己进行处理。
    判断线程的中断标志位:
        1. isInterrupted() 用于判断当前线程是否处于中断状态
        2. interrupted() 静态方法，判断当前线程是否处于中断状态，同时中断标志位改为false,重置
        3. 方法如果抛出InterruptedException,线程的中断标志位会被复位成false，需要在catch块中
           再次发起中断操作interrupt()
