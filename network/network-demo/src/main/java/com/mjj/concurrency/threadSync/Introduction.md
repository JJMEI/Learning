    Java同步机制
    为什么要同步?
    Synchronized内置锁
        1. 对象锁，锁的是对象实例
        2. 类锁，锁的是每个类的Class对象，每个类的Class对象在虚拟机中只有一个，因此类锁也只有一个
        3. synchronized的关键之不能被继承，父类的同步方法被重写，也需要显示的加上synchronized，否则就是
           不是同步方法
        4. 同步方法
            4.1 实例同步方法 锁是当前调用的对象
            4.2 静态同步方法 锁是当前类的Class对象
            
        synchronized 既保证了线程之间的可见性，也保证了操作的原子性 因此是线程安全的
        volatile 关键字只能保证线程之间的可见性，并不能线程操作的原子性
